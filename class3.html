Objects

Objects let us store a collection of properties in key/value pairs.


    var objectName = {
      propertyName: propertyValue,
      propertyName: propertyValue,
      ...
    };
  

      var aboutMe = {
      hometown: 'Los Angeles, CA',
      hair: 'Dark Brown',
      likes: ['playing guitar', 'code'],
      birthday: {
        month: 4, 
        day: 3
      }
    };
    // keys: hometown, hair, likes, birthday
    // values: string, string, array, object

Programmers use a lot of name/value pairs

  • HTML- uses attribute names and values
  • CSS- uses property names and values

    • in JS

  • variables- have a name and you can assign it a value
  • arrays- have a name and a group of ordered values
  • functions- have a name, and a value that is a set of statements to run if the function is called
  • objects- consist of name/value pairs(names are referred to as 'keys')

Create an Object(Literal Notation)

  1. the code begins like any variable, with the var keyword followed by a variable name
  2. after the equals sign, you add curly braces.
  3. inside of the curly braces, you have name: value pairs
  4. each property begins with a key, followed by a colon, and then the value.
  5. each property definition ends with a comma, except for the last one
  6. the closing curly brace is followed by a semi-colon

*** A property name is always a string so you don't have to put the names in quotes

Accessing Objects

You can retrieve values using "dot notation"

  
    var aboutMe = {
      hometown: 'Los Angeles, CA',
      hair: 'Dark Brown'
    };
    var myHometown = aboutMe.hometown;

Or using "bracket notation" (like arrays). The difference is that in the brackets, you add the property name in quotes, instead of a number

  var myHair = aboutMe['hair'];

retrieve values with objectName.name or objectName['name']

Changing Objects

You can use dot or bracket notation to change properties

  var aboutMe = {
  hometown: 'Los Angeles, CA',
  hair: 'Dark Brown'
};
aboutMe.hair = 'blue'; // will change my hair color to 'blue'

Add new properties

  aboutMe.gender = 'female'; 
// will add a gender:'female' the name:value pair to the object aboutMe

Or delete them

  delete aboutMe.gender;

Arrays of Objects

Since arrays can hold any data type, they can also hold objects in order 

  var myCats = [
  {
    name: 'Lizzie',
    age: 18
  },
  {
    name: 'Daemon',
    age: 1
  }
];

for (var i = 0; i < myCats.length; i++) {
  var myCat = myCats[i];
  console.log(myCat.name + ' is ' + myCat.age + ' years old.');
  // will output Lizzie is 18 years old.
  // then will output Daemon is 1 years old.
}

Objects in Functions

Just like other data types, an object can be passed into a function:

  var lizzieTheCat = {
  age: 18,
  furColor: 'grey',
  likes: ['catnip', 'milk'], // an array
  birthday: {month: 7, day: 17, year: 1996} // an object
};

function describeCat(cat) {
  console.log('This cat is ' + cat.age + ' years old with ' + 
  cat.furColor + ' fur.');
  // will output: This cat is 18 years old with grey fur.
}

describeCat(lizzieTheCat); 
// calls the function with the argument as the object LizzieTheCat

Object methods

Objects can also hold functions.

  
    var lizzieTheCat = {
      age: 18,
      furColor: 'grey',
      meow: function() {
        console.log('meowww'); 
      },
      eat: function(food) {
        console.log('Yum, I love ' + food);
      },
    };

Call object methods using dot notation.

  
    lizzieTheCat.meow(); // output will console.log: meoww
    lizzieTheCat.eat('cuban sandwiches'); 
    // output will console.log: Yum, I love cuban sandwiches

Constructor Notation

  function Animal(age, color) {
  this.age: age;
  this.color: color;
  this.noise: function(noise) {
    console.log(noise, noise);
  };
  this.food: function(food) {
    console.log('Yum! I love ' + food + '.');
  };
}
// create instances of the object using the constructor function
// Animal(18, 'grey', 'meow', 'tuna')
var lizzieTheCat = new Animal(18, 'grey', 'meow', 'tuna');
var dieselThePuppy = new Animal(1, 'white', 'ruff', 'carrots');

Things to Notice:

  • Use this function as a template for creating multiple objects
  • The function name is capitalized
  • The new keyword tells JS to create a new object

This

The keyword this references the element that the user has just interacted with:

  function Animal(age, color) {
  this.age: age; // this refers to the containing object
  this.color: color; // ends in a semi-colon
  this.noise: function(noise) {
    console.log(noise, noise);
  };
  this.food: function(food) {
    console.log('Yum! I love ' + food + '.');
  };
}
var lizzieTheCat = new Animal(18, 'grey', 'meow', 'tuna'); // arguments for the function
var dieselThePuppy = new Animal(1, 'white', 'ruff', 'carrots');

It is used instead of the object name to indicate that the property or method belongs to the obejct that 'this' function creates.

Object Prototypes

  function Animal(age, color) {
  this.age: age;
  this.color: color;
}
Animal.prototype.noise = function(noise) {
  console.log(noise, noise);
};
Animal.prototype.food: function(food) {
  console.log('Yum! I love ' + food + '.');
};
var lizzieTheCat = new Animal(18, 'grey', 'meow', 'tuna');
var dieselThePuppy = new Animal(1, 'white', 'ruff', 'carrots');
  • Prototypes help keep your code dry & help with performance
  • Objects can have prototype properties as well as prototype methods
    • Animal.prototype.lovable = true;

IDs vs. Classes

ID -- Like your driver's license, you should only apply an ID to one element on a webpage. For instance, a webpage only has one header or one footer.


Use "#" to tell CSS "this is an id."

  
    <h1 id='header'>We write the best JavaScript code!</h1>

    // in your main.css file, you can change the styles with: 
    #header {
      color: #660000;
      font-size: 3em;
    }

Class -- As you can take many classes in order to learn how to program, multiple elements can have the same class. For instance, there can be many warnings on one webpage.


The "." is how you tell CSS "this is a class name."

  
  <p class='warning'>
    This product is for puppies that are older than 1.
  </p>

  // in your main.css file, you can change the styles with: 
  .warning {
    color: #FF0000;
    font-weight: bold;
  }

The DOM Tree

We model the nested structure of an HTML page with the DOM (Document Object Model) Tree. The browser makes a "map" of all the elements on a page.


You can think of the DOM like a family tree. It shows how each child node in a website is connected to its parent node.

A node is any element on a page, beginning with the html tag (the 'oldest' parent).

The DOM: Sample Code

            <style>

                 color: red;
                }
            </style>

            <h1>My Page</h1>
            <p>Hello World!</p>
            <img src="http://placekitten.com/200/300" alt="cat"/>
        </body>

DOM Access

Your browser automatically builds a Document Object to store the DOM of a page.


To change what displays on a page:

  1. Find the DOM node and store it in a variable
  2. Use methods to manipulate the node

DOM Access: By Id

You can find nodes by the id using this method:

  document.getElementById('id');
// document refers to the 'document object'

To find the image with the id of 'kittenPic':

  <img id="kittenPic" src="http://placekitten.com/200/300" alt="cat"/>

We would use:

  var imgKitten = document.getElementById('kittenPic');
// assigns the img to imgKitten
// notice that 'Element' is singular, and the 'd' in 'Id' is lowercase
// as there can only be one 'kittenPic' ID on a page

DOM Access: By Tag Name

You can find HTML elements on a page using this method:

document.getElementsByTagName(tagName);

To find the li's on the page:

  <ul>
  <li>Daisy</li>
  <li>Tulip</li>
</ul>

We would use:

  var listItems = document.getElementsByTagName('li');
for (var i = 0; i < listItems.length; i++) {
  var listItem = listItems[i];
} // notice that 'ElementsByTagName' is plural  
// as there can be multiple elements with the same tag on a page

DOM Access: HTML 5

In newer browsers, you can use other methods too.

Available in IE9+, FF3.6+, Chrome 17+, Safari 5+:

  
    document.getElementsByClassName(className);

Available in IE8+, FF3.6+, Chrome 17+, Safari 5+:

  
    document.querySelector(cssQuery);
    document.querySelectorAll(cssQuery);

*** These are not used very often because they are not accessible/usable on every browser.

getElement vs. getElements

Any method that starts with "getElement" will return a single node.

  document.getElementById('uniqueID'); //returns a single node

Any method that starts with "getElements" will return an collection(like an array) of nodes. To modify a single node, you will need to use bracket notation to select the correct one.

  document.getElementsByTagName('p'); //returns multiple nodes
// to select a single paragraph use the tag name ('p') 
// followed by the index number in a bracket
var specficParagraph = document.getElementsByTagName('p')[2];
// assigns the third paragraph to the variable specificParagraph

DOM Nodes: Attributes

You can access and change attributes of DOM nodes using dot notation.

To change this element:

  <img id="kittenPic" src="http://placekitten.com/200/300" alt="cat"/>

We could change the src attribute this way:

  // assigns #kittenPic to the var imgKitten
// this allows you to access the image attributes via dot notation
var imgKitten = document.getElementById('kittenPic');
// assigns the old kittenPic src to the variable oldSrc
var oldSrc = imgKitten.src;
// changes the KittenPic src to a new url
imgKitten.src = 'http://placekitten.com/100/500';

***On the opening HTML element- the equal sign and quotes tell you that you are looking at an attribute

DOM Nodes: Getting and Setting Attributes

You can also use getAttribute/setAttribute to get and change the attributes of DOM nodes:

  <img id="kittenPic" src="http://placekitten.com/200/300" alt="cat"/>

We would change the src attribute this way:

  // assigns #kittenPic to the variable imgKitten
var imgKitten = document.getElementById('kittenPic');
// assigns the kittenPic src to the variable oldSrc
var oldSrc = imgKitten.getAttribute('src');
// changes the KittenPic src to a new url
imgKitten.setAttribute('src', 'http://placekitten.com/100/500');

hasAttribute will check to see whether an element has a particular attribute = true/false

DOM Nodes: Getting and Setting Attributes(cont)

You can use className to add to or update an element's class name:

<img id="kittenPic" src="http://placekitten.com/200/300" alt="cat"/>

We would change the src attribute this way:

  // assigns #kittenPic to the variable imgKitten
var imgKitten = document.getElementById('kittenPic');
// adds the class name 'kitten' to the #kittenPic
imgKitten.className('kitten');

removeAttribute will let you remove a particular attribute from an element

*** you might want to check if the attribute exists using hasAttribute first

DOM Nodes: Styles

You can change the css on a page using the style tag

On a CSS stylesheet, you can change the CSS this way:

body { color: red; } // changes the text color to red

The CSS link tag:

  <head>
    <link rel="stylesheet" type="text/css" href="main.css">
</head>

In JavaScript, you would use the style tag:

  var pageNode = document.getElementsByTagName('body')[0];
pageNode.style.color = 'red';

DOM Nodes: More Styles

Change any CSS property with a "-" to camelCase, and be sure to include a unit on any number

To make this CSS:

  
    body {
      background-color: pink;
      padding-top: 10px;
    }

Use this JavaScript:

  
    var pageNode = document.getElementsByTagName('body')[0];
    pageNode.style.backgroundColor = 'pink';
    pageNode.style.paddingTop = '10px';

DOM innerHTML

Each DOM node has an innerHTML property with the HTML of all its children. You can use the property to view or change the HTML of a node.

For example, you can overwrite the entire body:

  var pageNode = document.getElementsByTagName('body')[0];
pageNode.innerHTML = 
'<h1>Oh Noes!</h1> <p>I just changed the whole page!</p>'
// the '=' assigns the new content to the page
// this will replace all of the other content

Or just add new content to the end of a page:

  pageNode.innerHTML += 
    '<p>...just adding this bit to the end of the page.</p>';
// use the '+=' assignment to add content to the page
// this will just add, not replace the content on a page

DOM innerHTML continued

You can also target a particular element

To fill this HTML element:

  
    <p id="warning"></p>

We can select the node and modify it:

  
    // assigns the #warning paragraph to the variable warningParagraph
    var warningParagraph = document.getElementById('warning');
    // will put 'Danger Will Robinson' into the #warning paragraph
    warningParagraph.innerHTML = 'Danger Will Robinson!';

Creating New Nodes

The document object also provides ways to create nodes from scratch:

  
    document.createElement('tagName'); // will create a new element
    document.createTextNode('text'); // will create new text
    // will append the child to the parent
    document.appendChild(childToAppend);

Creating New Nodes: Sample Code

  // assigns the body to the variable pageNode
var pageNode = document.getElementsByTagName('body')[0];

// creates a new image elemnt
var newImg = document.createElement('img');
// assigns this src to the new image
newImg.src = 'http://placekitten.com/400/300';
// adds a border style to the image
newImg.style.border = '1px solid black';
// appends the new image to the body via the variable pageNode
pageNode.appendChild(newImg);

  // creates a new paragraph element and assigns it to var newParagraph
var newParagraph = document.createElement('p');
// creates the text 'Squee!' and assigns it to var paragraphText
var paragraphText = document.createTextNode('Squee!');
// appends the paragraphText to the var newParagraph
newParagraph.appendChild(paragraphText);
// appends the new paragraph to the body via the variable pageNode
pageNode.appendChild(newParagraph);

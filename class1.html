Quick HTML & CSS Refresher

HTML

  • The HTML gives the page structure and adds semantics
  • HTML elements are added to the content of a page to describe its structure
  • An element consists of its opening and closing tags, plus its content. (There are very few self-closing tags)
  • Opening tags can carry attributes, which tell us more about that element.
  • Attributes have a name and a value- like class or href

CSS

  • CSS uses rules to display content in the browser.
  • Each rule has a selector and a declaration.
  • The selector indicates which element the rule applies to
  • The declaration includes a property(the thing you want to change) and a value

What is JavaScript?

  • JavaScript is standardized by the "ECMAScript" specifications, which allows it to behave the same way in all applications.
  • JavaScript is a client-side scripting language. A browser reads the code and runs it directly.
  • JavaScript interfaces with HTML and CSS, adding basic intelligence and simple functionality to web sites.
  • With JavaScript, you can write code once and use it everywhere. Remember, you want DRY code (Don't Repeat Yourself).
  • JavaScript lets you build dynamic webpages that respond to input from users.

Client-side Programming

Client-side programming is the name for programs that are run on the Client, usually a web browser. The processing takes place on the end user's computer.


Uses

  • Make webpages interactive.
  • Make things happen dynamically.
  • Send requests and retrieve data from the server.

Server-side Programming

Server-side programming is the general name for programs which are run on the Server.


Uses

  • Process user input.
  • Displays pages.
  • Structures web applications.
  • Interacts with permanent storage (SQL, files).

JavaScript is a client-side scripting language

What is a Script?

A script is a series of instructions that a computer can follow to achieve a goal, like a recipe.

A computer solves tasks differently than humans, so you have to create your instructions so that the computer can solve the task programmatically.

Script Tags

You can add JavaScript to your HTML page. The script tag tells your browser the stuff inside is code, not content.

  
    <script type="text/javascript">
      CODE GOES HERE
    </script>

JavaScript Files

Just like in CSS, you can and should put the JavaScript into its own file. 

  
    <script src="js/main.js" type="text/javascript"></script>

Best Practice

  • If you keep the JS in a separate file, it means that the page will still work if the user cannot load the JS
  • You can also reuse the JS on multiple pages
  • Put the script src tag at the bottom of your HTML page, before your closing body tag.

Separating Instructions

A statement is an individual instruction that the computer should follow. After each individual statement, you must add a semicolon.

  
    <script>
      console.log('Hello World!');
      console.log('I am glad to meet you');
      console.log('I am fuzzy');
    </script>

A statement tells the JS interpreter when a step is over, indicating that it should move on to the next step.

Comments

You should leave comments in your code—notes that can help people read and understand your code, and that computers will ignore.

  
    <script>
      /*I can wrap long descriptions about how
      the code works on multiple lines
      like this*/

      //Or write short descriptions like this
      console.log('Hello World!'); 
    </script>

Variables


A variable is a place to store values

Variable Values


  • When you first create/declare a variable (with the var keyword), it does not have a value (it is undefined).
  • You can assign a value to a variable.
  • Variables can hold different types of information: like strings(text), integers(whole numbers), floats(decimals), booleans(true/false) or collections of data.
  • The value of the variable can change every time the code runs.

Declaring a Variable

To declare (create) a variable, just type the keyword "var" and the variable name.

  
    <script>
      var numberOfKittens;
    </script>

It is a good idea to give your variable a starting value. This is called initializing the variable.

  
    <script>
      var numberOfKittens = 5;
    </script>

Naming Variables


  • All variables start with lowercase letters.
  • The variable name is case-sensitive.
  • A new variable needs to have a unique, descriptive name.
  • Variable names need to start with a letter, $, or _.
  • Variable names can only be made of letters, numbers, $, or _.
  • reserved words- https://developer.mozilla.org/en-US/docs/Talk%3AJavaScript/Reference/Reserved_Words


Best Practice

  • It is best practice to use camelCase when there is more than one word.

Using a Variable

Once you have declared a variable, you can use it in your code. Just type the name of the variable.

  
    <script>
      var numberOfKittens = 5;
      console.log(numberOfKittens); // logs 5
    </script>

The Different Types of Variables

Numbers

Variables can be numbers, either integers (whole numbers) or floats (decimals).

  
    <script>
      var numberOfKittens = 5; // integer
      var cutenessRating = 9.6; // float
    </script>

The browser will automatically convert integers to floats if necessary

Numbers are Used for:


  • Calculators
  • Determining the size of a screen
  • Moving the position of an element on a page
  • Setting the amount of time an element should take to fade in

Arithmetic Operators

Once you have numbers, you can do math with them!

  
    <script>
      var numberOfKittens = 5;
      var numberOfPuppies = 4;
      var numberOfAnimals = numberOfKittens + numberOfPuppies;
    </script>

Strings

Variables can be strings, groups of characters. You put your string in single or double quotes.

                
                  <script>
                    var kittensName = 'Fluffy';
                  </script>

If you want to use a quote in your string, you'll need to "escape" it with a backslash. A backslash tells the interpreter that this is not the end of ths string.,,

                
                  <script>
                    console.log('I\'d like to use an apostrophe');
                  </script>

Strings are Used:


  • When working with any kind of text
  • To add new content to the page
  • They can contain HTML markup

String Operators

You can put strings together with a '+', the concatenation operator.

  
    <script>
      var kittensName = 'Fluffy';
      var fullName = kittensName + ' McDougal';
      console.log(fullName); //Outputs 'Fluffy McDougal'
    </script>

Best Practice

  • Notice the space before the ' McDougal'. Without that space, the console.log would output 'FluffyMcDougal'.

String Operators

You can also use += to add things to the end of a string.

  
    <script>
      var kittensName = 'Admiral';
      kittensName += ' Snuggles';
      console.log(kittensName); //Outputs 'Admiral Snuggles'
    </script>
  • kittensName += ' Snuggles'; is the same as kittensName = kittensName + ' Snuggles';

Mixing Numbers and Strings Together

  • When there are quotes around a number, it is a string, not a number.
  • You cannot perform addition operations on strings, it will just add the two strings together.
  • '7' + '9' = '79'

Functions


Functions are separable, reusable pieces of code.


You define a function once, and then you can use it multiple times.

How to Declare (Create) a Named Function


  • start with the keyword 'function'
  • followed by the functionName- named the ame way as variables
  • followed by parenthesis
  • followed by curly braces
  • inside the curly braces, you write the code that you want to execute
  • to execute a function you need to call it, with the functionName followed by parenthesis

Using Named Functions

First, declare the function.

  
    <script>
      function turtleFact() {
        console.log('A turtle\'s lower shell is called a plastron.');
      }
    </script>

Then, you can call it (use it) as many times as you want!

  
    <script>
      turtleFact();
    </script>

Parameters

Functions can accept input values, called parameters. Parameters act like variables in the function, and let us provide info to the function.

  
    <script>
      function callKitten(kittenName) {
        console.log('Come here, ' + kittenName + '!');
      }
      callKitten('Fluffy'); //outputs 'Come here, Fluffy!'

      function addNumbers(a, b) {
        console.log(a + b);
      }
      addNumbers(5,7); //outputs 12
      addNumbers(9,12); //outputs 21
    </script>

'kittenName' and 'a', 'b' are the parameters.

Arguments

When you call a function that has parameters, you specify the arguments (values) it should use in the parenthesis that follow its name.

  
    <script>
      function getArea(width, height){
        var area = width * height;
        console.log('<p>The area is ' + area + '</p>');
      }
      //Declare variables
      var wallWidth = 5;
      var wallHeight = 4;
      //Use them in functions
      getArea(wallWidth, wallHeight);
      getArea(3, 5);
    </script>

numberOfKittens and numberOfPuppies are the arguments.

Returning Values

You can have a function give you back a value to use later.

  
    <script>
      function square(num) {
        return num**;
      }
      console.log(square(4));   // outputs '16'.
      var squareOfFive = square(5); // will make squareOfFive equal 25.
    </script>

Return will immediately end a function. If there are subsequent statements in the function, they will not be run.

JS best practices

  • code should be readable
  • code should be consistent
  • always use semicolons to end a statement
  • define your functions before you call them

    • naming conventions

  • use camelCase
  • except with objects- capitalize the first word
  • always use var when declaring a variable
  • use descriptive variable and function names

    • brace styles

  • curly brace- starts on the same line as the keyword - ends at the same indentation
